# Potential Gaps for **Printbit**

1. Ink monitoring, on how the system / kiosk to the end users if there is low ink or out of ink. This is important to ensure that users are aware of the ink status and can take necessary actions to avoid printing issues.
2. Android and iOS for the **Print** method
3. How admin be notified if there is a problem with the system, kiosk machine, or printer. This is crucial for timely maintenance and to minimize downtime.
4. On how secure the payment (coin insertion) is.
5. What if the printer itself needs to replace or repair, how will the system handle it? Will there be a backup printer or a way to notify users of the issue?
6. User interface and user experience design for the kiosk, ensuring it is intuitive and easy to navigate for users of all ages and technical abilities.
7. About paper jam, is end-users that uses it has a refund action?
8. In **Print** method, what if users are not yet decided on the file(s) they want to print since it has a session?
9. **NO** user queueing feature
10. **NO** user surveilance feature, which could be important for security and monitoring purposes.
